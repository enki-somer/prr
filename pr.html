1,362 of 1,365 "done with no app data" jobs are installations — technicians for installations are not using the mobile app at all in large numbers. This is a process compliance gap, not a system error.

Why is installation action log coverage only 48.9%? The Odoo mobile field service app requires technicians to manually tap through each stage (Assign → On The Way → Ongoing → Done/Failed). Maintenance teams adopted this workflow early and follow it consistently (97.6% coverage). Installation teams, particularly in the KUT region which handles 3,024 WOs, often complete jobs without opening the app — the team leader marks the work order done directly in the back-office system, bypassing the mobile workflow entirely. The root cause is training and enforcement, not a system defect.

Why is technician name missing? Odoo's field service module assigns work orders to a team, not necessarily to an individual. For installation jobs, the team leader records completion without logging the individual technician who did the physical work. The field in Odoo (user_id) is optional on the work order form and was not enforced during the rollout period. This means individual-level performance tracking, accountability, and payroll reconciliation are not possible for 54.5% of installation records from the source system. This is a data entry policy gap, not a technical limitation.

Parsing the description template field recovered FAT data for 3,463 work orders — a massive analytical gain. For maintenance, FAT coverage jumped from 12.6% → 78.8%. System GPS coverage is 55.6% (5,868 WOs) for non-zero coordinates — the remaining 44.4% (4,686 WOs) contain the coordinate 0°,0° (Gulf of Guinea / Null Island), which is Odoo's default when no GPS lock was recorded. Only description-parsed GPS (68.1% of maintenance) represents real field-recorded site locations.

Why is FAT linkage missing for 42.5% of WOs? The fat_id field on a work order in Odoo is a voluntary lookup — field coordinators are not required to select a FAT when creating the job. During the early deployment period (Nov–Jan), data entry was faster and less structured. The FAT reference only became consistently used once the description template was standardised, which is why parsing the template text recovered 3,463 records.

Why is GPS showing 0°,0° (Null Island) for 44.4% of WOs? Odoo stores GPS as a decimal coordinate pair defaulting to 0.0, 0.0 when the field is not populated. The mobile app records GPS when a technician logs an action (assign, on-the-way, etc.). For installation jobs where the app was not used (48.9% of installations), no GPS was ever written — so the coordinate stays at the system default 0°,0°. This is not a map error; it is an empty field presented as a coordinate.
